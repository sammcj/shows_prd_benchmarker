Top 5 Gaps — and Why They Matter

These are not minor omissions. Each one has a concrete failure mode if left unaddressed.

1
PRD-073 Critical
Export My Data: .zip with JSON backup, ISO-8601 dates

The plan includes an ExportButton component but specifies nothing about the export format. The PRD requires a .zip containing JSON with ISO-8601 dates. "Your data is yours" is a cross-cutting principle — data portability is a first-class concern. Without the format spec, the export feature is an empty shell: the component exists but the contract is undefined.

2
PRD-082 Important
Non-my field merge: selectFirstNonEmpty

The storage schema mandates that catalog refreshes never overwrite a non-empty stored value with an empty one. Without this, a metadata API returning partial data silently blanks out posters, overviews, or genre lists. Users see broken tiles with no indication data was lost and no recovery path. This is the most likely silent corruption vector.

3
PRD-083 Important
My-field merge by timestamp (newer wins)

The PRD requires per-field timestamp conflict resolution. The plan uses a single updated_at on MyData instead of field-specific timestamps. Any sync or concurrent-edit scenario risks overwriting a user's intentional change with a stale value — directly violating the "user's version takes precedence" rule.

4
PRD-085 Important
Data continuity: preserve libraries across updates

No data model versioning, no AppMetadata entity, no migration strategy. In a benchmark where schemas will iterate, this is a structural omission. Users lose their collection after an app update with no recovery mechanism — exactly the scenario the PRD explicitly prohibits.

5
PRD-086 Important
Unresolvable recs: graceful degradation

AI prompts require real catalog matches, but there is no fallback when resolution fails. The PRD specifies unresolvable titles should appear non-interactive or hand off to Search. Without this, a hallucinated recommendation becomes a dead end or runtime error instead of a graceful degradation.

Remediation Path

Three categories of work to close the gaps. All additive — no architectural rework needed.

New Plan Sections

Data integrity gaps need new sections, not just more detail on existing ones.

  • selectFirstNonEmpty merge policy for catalog data
  • Per-field timestamp schema (replace single updated_at)
  • Re-adding flow preserving existing My Data
  • AppMetadata entity for data model versioning
  • Unresolvable rec fallback pathway

Settings Completeness

Each PRD setting needs a corresponding component and data model entry.

  • Font size / readability control
  • Username setting with sync support
  • Search on Launch toggle
  • AI model selection (not just persona)
  • Explicit API key management UI

Missing Edge-Case Specs

UX details and failure modes that need concrete acceptance criteria.

  • Ask welcome view with 6 starter prompts
  • Collection 4-group hierarchy with tile sizes
  • Genre, decade, score, "No tags" filters
  • AI parsing failure: retry + fallback
  • Header carousel fallback behaviour
